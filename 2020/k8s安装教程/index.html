K8S安装教程

K8S 2020-10-01 2081 字 5 分钟 次

K8S集群安装文档

一、基础环境配置

  • 物理机环境配置：
    1. CPU：i7-6700 3.6GHZ
    2. 内存16G
    3. 硬盘1T
    4. 操作系统：win10 64位
  • 利用VMware创建四台虚拟机：
    1. CPU : 2（master）1（slave)
    2. 内存：1G
    3. 硬盘：20G
    4. 网络配置：NAT
    5. 操作系统：CentOS 7

二、虚拟机配置：

  • 安装JDK：

    1. 首先输入java -version，如果显示java版本信息则系统中JDK已经安装如下所示：

      [root@xxx ~]# java -version
openjdk version "1.8.0_222"
OpenJDK Runtime Environment (build 1.8.0_222-b10)
OpenJDK 64-Bit Server VM (build 25.222-b10, mixed mode)

      直接跳过JDK安装

    2. 使用yum源安装jdk1.8最新版本：

      sudo yum install java-1.8.0-openjdk* -y

  • 配置网络：

    1. 配置IP地址（实验室集群安装时可以忽略此步）：

      系统安装后，此时如果主机是通过路由器自动获取IP地址上网的，那么虚拟机由于选择了NAT模式，应该也是可以联外网的。但是，由于没有将虚拟机的网卡激活，此时可能无法联网。因此需要先编辑ifcfg-ens33文件（名字不同虚拟机可能会有所不同）激活网卡：

      cd /etc/sysconfig/network-scripts/
vi ifcfg-ens33

      将ONBOOT=no 改为 yes，BOOTPROTO=dhcp修改为BOOTPROTO=static，然后修改IP地址以及DNS地址：

      IPADDR=192.168.5.101
NETMASK=255.255.255.0
GATEWAY=192.168.5.2

      然后执行命令 service network restart重启网络服务。

      成功之后输入ifconfig来查看虚拟机的IP地址，如果虚拟机采用了最小安装此时ifconfig是无法使用的，此时输入yum install net-tools,即可利用ifconfig来查看网络：

    2. 配置yum源 不建议使用CentOS 7 自带的yum源，因为安装软件时会很慢。这里，我们使用阿里云的源予以替换，执行如下命令，替换文件/etc/yum.repos.d/CentOS-Base.repo：

      sudo yum clean all
sudo wget -O /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo 
sudo yum makecache

    3. 修改hostname

      安装集群时Master和Slave节点不好区分，可以配置Hostname来区分Master和Slave。输入sudo vim /etc/hostname 将内容修改为Master或者Slave。

    4. 关闭防火墙 防火墙一定要提前关闭，否则在后续安装K8S集群的时候是个trouble maker。执行下面语句关闭，并禁用开机启动：systemctl stop firewalld & systemctl disable firewalld

    5. 关闭Swap（放置内存交换导致集群性能下降）： 类似ElasticSearch集群，在安装K8S集群时，Linux的Swap内存交换机制是一定要关闭的，否则会因为内存交换而影响性能以及稳定性。这里，我们可以提前进行设置：

      执行 swapoff -a可临时关闭，但系统重启后恢复 编辑/etc/fstab，输入sudo vim /etc/fstab，注释掉包含swap的那一行即可，重启后可永久关闭，如下所示：

      /dev/mapper/centos-root/xfs defaults 0 0
UUID=20ca01ff-c5eb-47bc-99a0-6527b8cb246/boot  xfs  defaults  0 0
# /dev/mapper/centos-swap swap

      关闭成功后，使用top命令查看，如下图所示表示正常：

      到此，安装K8S之前所有虚拟机的基本准备工作已经完成，接下来让我们安装K8S

  • 安装K8S：
    1. 安装Docker：

      安装K8S必须要先安装Docker。这里，我们使用yum方式安装Docker社区最新版，执行以下命令，安装最新版Docker：

      sudo yum install docker-ce -y

      在实验室集群中安装出现问题：

      安装时发现会出现No package docker-ce available.的问题，应该是因为yum源的镜像版本太老导致的，所以安装 了yum-utils，它提供了 yum-config-manager，可用来管理yum源，进而个更新yum源：

      sudo yum install -y yum-utils

      添加新的yum源：

      sudo yum-config-manager --add-repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

      更新索引：

      sudo yum makecache fast

      再次安装docker：

      sudo yum install -y docker-ce

      安装成功！

      启动Docker服务并激活开机启动：

      systemctl start docker & systemctl enable docker

      验证一下，是否成功，输入：

      docker run hello-world

    2. 安装K8S组件：

      由于自版本1.6.0起官方的仓库无法使用，所以使用案阿里云的仓库，添加kubernetes.repo仓库：

      cd /etc/yum.repos.d/
sudo vi kubernetes.repo

      将以下内容复制到kubernetes.repo中：

      [kubernetes]
      
name=Kubernetes
      
baseurl=http://mirrors.aliyun.com/kubernetes/yum/repos/kubernetes-el7-x86_64
enabled=1
gpgcheck=0
repo_gpgcheck=0
gpgkey=http://mirrors.aliyun.com/kubernetes/yum/doc/yum-key.gpg
        http://mirrors.aliyun.com/kubernetes/yum/doc/rpm-package-key.gpg

      安装K8S组件

      执行以下命令安装kubelet、kubeadm、kubectl：

      sudo yum install -y kubelet kubeadm kubectl kubernetes-cni
    3. 启动Kubelet：

      sudo systemctl enable kubelet && systemctl start kubelet（slave节点无需启动kubelet）

    4. 配置hosts
      sudo vi /etc/hosts

      添加IP地址与hostname的映射，即在文件最后一行添加

      192.168.0.209 vm0
    5. 配置docker网络

      安装k8s前还需做一步容器网络配置，执行以下命令

      vi /etc/sysctl.conf

      打开后添加以下两行

      net.bridge.bridge-nf-call-ip6tables = 1
net.bridge.bridge-nf-call-iptables = 1

      之后再运行命令：

      sysctl -p

      以上命无论时Masker还是Slave节点都需要执行

    6. 接下来部署master节点：
      sudo kubeadm init --image-repository=registry.aliyuncs.com/google_containers --pod-network-cidr=10.244.0.0/16

      用docker下载flannel文件

      docker pull quay.io/coreos/flannel:v0.11.0-amd64

      执行命令：

      kubectl edit cm coredns -n kube-system

      按照以上提示，执行

      mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

      然后，去官网https://kubernetes.io/docs/concepts/cluster-administration/addons/找到flannel相关命令执行，在这里直接运行以下命令

      kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml

      安装完成后，运行以下命令查看

      kubectl get pod -n kube-system

    7. 将剩余slave加入集群中：

      配置完成后，在master节点下查看当前

      kubeadm token create --print-join-command

      复制以上命令各个slave节点当中即可将节点部署到k8s集群中，搭建完成后，在主节点上运行

      kubectl get nodes

​ 出现以下内容，说明集群部署安装成功

systemctl restart kubelet
