异常流量检测相关文献调研

异常流量检测相关 2020-12-24 3983 字 8 分钟 次

首先是一篇关于异常流量检测的综述：

A comprehensive survey on network anomaly detection

年份：2018

来源：Telecommunication Systems

  • 主要研究问题：对网络异常检测做了一个综述
  • 主要内容：

首先作者对异常流量进行了一下分类：

之后作者对异常流量的特征提取工具进行了一下分类们主要可以分为两个方面：

  1. tcpdump：对原始包进行分析
  2. IP流量：针对网络中的每一条流量进行分析（Netflow探针）

作者提出流量与处理数据包分析相比

  1. 可以大大节省分析成本
  2. 减少隐私问题，原始数据包中由一些敏感数据

同时基于流量，由于流量时端到端的，所以仅基于流量无法提供整个网络的可靠试图，当网络拓扑发生变化而网络流的属性不变时无法检测到。除此以外端到端流量得属性随时会变化，因此理解流量也很困难。

  • 此外作者还讨论了目前IDS得一些缺陷：

目前IDS很难检测到入侵，仅仅能检测到入侵得证据，还无法做到实时检测。

IDS主要分为两中一种是基于主机得IDS另一种是基于网络的IDS，一个部署在主机上，一个在网络的主干中。

  • 作者根据大量异常检测方法的对比提出：
    1. 关于正常流量的概念是尤其重要的：目前的许多工作都是基于流量异常的不同定义进行工作的，目前还没有一个对异常流量的公认的定义。
    2. 异常是不断变化的，着包括两方面的理解：
      • 同一场景下的流量（正常或者异常）在不断变化，如何适应同一场景中的流量变化
      • 不同场景的流量（正常或者异常）不相同，如何用一套模型适用所有的场景。

​ 因此，IDS需要不断更新以适应这些更改，而这并非易事。

  1. 每当基于异常的IDS检测到未知攻击并加以解决时，都需要使用这些新数据来更新异常流量数据库，在不影响性能和产生冲突的情况下进行更新也是一个挑战
  2. 异常标注数据少， 缺乏可以模拟准确网络环境的可靠的公共标准数据集仍然是一个问题。
  3. 漏报率误报率高，使用更加精确的检测技术来实现异常流量的精确识别仍然是一个挑战
  4. 无法做到实时监控流量信息，没有信息就无法分析，虽然IDS基本上不能做到实时检测，但仍需要在入侵之后检测到入侵的证据
  • 作者给出了目前异常流量检测领域还在关心的问题：

可以看到目前大多的60％的受访者认为标准数据集问题是关键，缺乏标准的和更新的数据集来模拟真实环境并包含异常标签的问题是文献从业人员最关心的问题。

针对异常数据标注较少问题的方法

A Method of Few-Shot Network Intrusion Detection Based on Meta-Learning Framework

年份：2020

来源：IEEE TRANSACTIONS ON INFORMATION FORENSICS AND SECURITY (TIFS, CCF A)

  • 主要研究问题：

这篇文章的研究问题就是网络异常中的小样本检测

  • 研究动机（Motivation）：
    1. 目前，网络空间环境在不断变化，新型攻击也在不断发生。在某些情况下，例如zero-day，安全机构只能拦截有限数量的恶意样本。 因此，需要小样本检测。
    2. 此外异常标注数据少的特征在各种异常检测数据集中都有体现。
  • 挑战：

针对小样本来训练正确率高的网络异常检测模型比较困难

  • 创新点：
    1. 这篇文章是将meta-learning和fewshot-leanring引入网络异常检测领域的第一篇文章
    2. 此方法可用于直接从原始流量中学习网络流量分类的先验知识
    3. 提出了一种基于实际网络流量构建数据集的方法，可以用于小样本学习
    4. 证明了所提出的网络入侵检测方法是通用的，并且不仅限于特定的攻击类型。 使用所提出的方法，可以基于学习到的先验知识来检测未经训练的数据集中有限带标记的新型样本。
  • 模型描述：
    • 特征提取：这篇文章是利用原始流量数据中的前N个字节作为特征
    • 将原始数据输入后，作者利用FC-Net网络，来对输入的两个结构相同的样本进行一个比较。得出两个流量之间的差异，给定两个样本的增量得分。增量分数是一个实数，表示两个输入数据流之间的差异。F-net用于生成两个样本的也正，类似（encoder-decoder），c-net用于检测，输出的是两个样本之间的增量score

  • 与传统的原学习不相同，他的网络输出的不是label，是一个增量值，首先训练集的label已知，他从测试集中选取一个样本。然后通过网络的参数计算测试样本和正常训练样本，异常训练样本之间的增量值，也就是差异值。然后选择差异最小的作为predit，根据公式

计算错误率，然后更新loss。

  • 实验结果：
    1. 基于三个公开数据集：ISCX2012，CICIDS2017，CICIDS2018。正确率最高可以达到97.87%，最低94.33%，（注：作者提出如果才采样达到25%的话，正确率会降低，因为同一个算法无法很好的适应三个数据集）
    2. 作者还将FC-Net和ResNet-18进行了对比，由于ResNet-18直接训练的话会存在异常样本少的问题，因此可能会训练不到异常的样本，所以FC-Net的效果要比普通的CNN效果好。
  • 缺点：
    1. 没有考虑网络拓扑中的异常，仅仅基于流量的属性
    2. 仍然不能自适应网络中的流量分布
    3. 关于为什么要使用增量得分这种模型，作者并没有给出一个明确的解释，如果使用MAML，检测效果会不会相同。
    4. 更多的场景中根本就没有异常标记的流量，只有一些正常流量的日志。

目前仅仅针对恶意样本标注少进行有监督学习的文章不多，大多的方法还是做无监督，不过meta-learning提供了一种检测出来unknown流量的可能。

基于图的方法：

基于图神经网络的动态网络异常检测算法

年份：2020

来源：软件学报

  • 主要研究问题：

利用Graph Embeeding的方法对数据进行无监督学习，从而进行异常检测，提出一种面向动态网络无监督表示学习算法

  • 研究动机（Motivation）：
    1. 如何在动态图种结合图的结构特征和图种节点的属性特征
    2. 针对网络中异常数据标注少采用无监督的方式进行训练
  • 挑战：
  • 创新点：
    1. 提出一种Graph Embedding的方式，同时提取到了图的结构信息，和由流量构成的线图中流量属性的聚合表示，通过两种embedding的结合来实现异常的检测
    2. 基于互信息最大化的网络表示学习，就是一种sample的方法，不同于deepwalk和sdne等等。
  • 模型描述：

这篇文章本质上就是利用了很多已有的算法来实现异常流量的检测，创新就是提出了一个架构，一个新的Graph Embedding的采样方法，并没有针对什么瓶颈和场景。

自动编码器（近年来文章较多）

GEE: A Gradient-based Explainable Variational Autoencoder for Network Anomaly Detection

年份：2019

来源：2019 IEEE Conference on Communications and Network Security

  • 主要研究问题：
    1. 以前的无论是统计学还是机器学习的方法都需要大量标注数据进行训练，而这样的数据集在实际中很少存在
    2. 以往的模型无法对异常的现象进行解释，作者提出了一种用于异常解释的基于梯度的指纹技术
    3. 不仅仅只针对恶意攻击，优于故障，配置错误等网络错误也可以识别出来
  • 研究动机（Motivation）：
    1. 异常标注数据少
    2. 以往的模型无法对异常现象进行解释
    3. 以往的方法需要用户一定的专家知识来选择流量的特征，这样对用户要求较高，而且无法做到自适应。所以需要一种自动编码器将异常现象编码成指纹，来自动的提取到特征。
  • 挑战：
    1. 由于影响网络的因素众多，包括面向用户的服务调整，产品的上线下线等等，此外用户的行为是不确定的，因此用户的行为也会影响网络。
    2. 由于目前网络容量变得越来越高，因此在高通量的网络当中捕获，存储和处理原始流量是不太可能的（但是由于DPDK的快速发展，目前针对一般通量的网络还是可以做到原始数据包的抓去）。
    3. 作者提出NetFlow经常是基于采样的原始包提取的，因此会丢失一些关键信息。所以异常检测解决方案必须处理有损信息
    4. 作者提出IDS的主要作用是帮助安全分析师分析网络中存在的安全问题，不能仅仅给出一个流量是不是恶意的，流量产生的时间，可疑流的集合等等信息，会对分析师有很大的帮助。
  • 创新点：
    1. 作者提出了一种基于VAE（自动变分编码器）的网络异常检测框架GEE，可以在NetFlow提取到的流量中进行自动的特征选择。
    2. 该框架在数据大小和特征维度上均可扩展
    3. 我们开发了一个基于梯度的框架来解释检测到的异常，并识别导致异常的主要特征
    4. 首次尝试使用VAE模型生成的梯度来解释网络异常
  • 模型描述：

与AE（自动编码器相比）VEA更加灵活和健壮，并且基于VAE的梯度可以对机器学习模型进行解释。下图简单介绍一下AE的组成。

​ VAE与AE不通VAE是一个生成模型，将潜在表示层视作基于输入的随机变量。VAE的encoder输出的是 表示层的预计的方差和均值，在根据均值和方差构建高斯分布，在进行采样作为输出进行decoder。 VAE的示意图如下：

​ VAE介绍之后作者提出了他们的模型GEE :

​ 首先，从NetFlow数据中提取重要信息, 提取到了一组包含53个特征信息的聚合特征。在进行去噪和 归一化后，作为VAE的输入，以学习网络的正常行为。 随后，根据生成的流量和原始流量之间的均方 差作为指标来识别异常。 此外作者还提出基于特征中每个元素在监测的的梯度，我们可以知道一条异 常流哪个特征是认为是异常的，基于特征的相似度判断还可以得出根据行为相似的流量组。

  • 实验结果：

数据集：

UGR16数据集，数据中包含真实数据，不是像CICIDS一样是模拟的。另一个数据集是MAWI同样是真实数据，而且每天更新。UGR中的恶意流量类型：

  1. Low-rate DoS
  2. Port scanning
  3. Botnet
  4. Spam
  5. Blacklist

实验结果：

首先基于VAE自定免罚后的向量能有效的区分出来正常和异常，作者提出的GEE模型相比于AE和VAE识别的效果要好一些。基于训练后流量中53个特征的梯度形成指纹可以更好的监测异常流量，并且可以基于相似度匹配识别新型的异常流量。

  • 缺点：
    1. 我觉得这篇创新点不足，VAE不是作者自己提出的，作者提出的模型GEE仅仅是基于VAE进行重构的特征，进行相似度的计算，与愿流量特征相似度较低的被认为是异常流量。基于特征的梯度构建异常的流量指纹是本文仅有的创新点。
    2. 无法自适应流量变化
    3. 论文中关于53个特征选择的依据没有介绍
    4. 没有考虑结构特征
